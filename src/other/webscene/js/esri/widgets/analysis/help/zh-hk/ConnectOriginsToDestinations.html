連接起點與終點

“連接起始點到目的地”用於測量點對之間的行駛時間或距離。 該工具可報告直線距離、基於行進模式的距離（例如步行或駕車）或基於行進模式的行程持續時間。 使用此工具，您可以執行以下操作：

  • 計算通勤者在從家到單位的行程中所用的總距離或時間。
  • 測量客戶到您的商店購物所行駛的距離。 利用這一資訊來確定市場覆蓋範圍，特別是當您鎖定廣告活動或選擇新的商店位置時。
  • 為您的車隊計算預期的行駛里程。 此後，執行“範圍內匯總”工具來按州或其他區域報告里程。

您提供起點和終點後，此工具便會在成對的起始點和目的地之間返回含有各路線和測量值的圖層。

如果勾選 使用當前地圖範圍，則在分析中僅考慮當前地圖範圍中可見的圖徵。 如果未勾選，則考慮輸入圖層中的所有圖徵，包括超出當前地圖範圍的圖徵。

選擇目的地圖層

選擇代表該目的地或起始點應該連接的目的地的圖層。 此工具最多支持 10,000 個起始點和 10,000 個目的地。

如果您選擇的圖層僅有一個目的地，則所有的起始點都會連接到該目的地。

如果有一個起始點和多個目的地，則此工具會將該起始點連接到每一個目的地。

如果有多個起始點和多個目的地，您需要具有兩個描述起始點和目的地對的欄位： 起始點中的 ID 欄位和 目的地中的匹配 ID 欄位。 此工具在分析期間連接起始點和目的地對。 例如，假設您有兩個 ID 欄位值分別為 Rosie 和 Harold 的起始點以及兩個帶有相同 ID 值的目的地。 此 Rosie 起始點將連接到 Rosie 目的地，並且 Harold 起始點將連接到 Harold 目的地。 不會執行其他連接或測量。

每一圖層內的 ID 值必須是唯一的；例如，目的地圖層不能有兩個 Rosie 值。 起始點和目的地中的值必須完全匹配才能配對； Harold 不會連接到拼寫錯誤的值， Harodl。

測量

選擇使用直線距離還是行進模式連接起始點和目的地。 可用的行進模式選項包括行駛距離、行駛時間、貨運距離、貨運時間、步行距離和步行時間。 輸出路徑將沿著道路或人行道並支持適用的規則，例如駕車或貨運的單行道和違規轉彎。

選擇 行駛時間時，您可以選擇行駛時間的測量方式。

  • 如果取消勾選 使用交通流量，則會基於固定速度尋找最快路徑。

    此選項適用於常規行駛時間的測量，而不適合於針對某一特定啟程時間使用變化的交通狀況計算行駛時間。

  • 勾選 使用交通流量以便基於給定的開始時間和變化的交通狀況尋找最快路徑。 透過選擇 即時交通流量選項，可將起始點的啟程時間設置為當前時間，並透過來自感應器和數據源的當前和預測車流速度建立路徑。 預測車流速度可透過參考即時車流速度、歷史速度和當前事件（如天氣）來計算。 可預測未來12 小時的車流速度，因此您透過移動滑桿設置的出發時間最多可提前 12 個小時。

    透過這些即時交通流量設置，您可以尋找現在出發所花費的行駛時間，一個小時後出發所花費的行駛時間等資訊。

  • 透過勾選 使用交通流量並選擇 基於以下典型狀況的交通流量所對應的日期和時間，該工具可根據一周內每五分鐘執行一次測速的歷史平均速度來尋找最快路徑。 結果中雖然包含交通流量，但會移除當前交通狀況和事件的影響，可能會與正常情況有很大的差別。

    您可以使用這些歷史交通流量設置解決以下問題，例如，“在星期三上午 11:30 出發通常需要花費多長時間到達這些目的地？”

    設定時間採用的是起始點所處位置的時區。 因此如果您將時間設置為上午 8:00，並採用兩個起始點 - 一個在紐約市，一個在洛杉磯，則會分別生成對應於啟程時間為東部時間上午 8:00 和太平洋時間上午 8:00 尋找最近位置。

請注意，車輛駛離起始點的同時，相應的時間流逝和交通狀況會隨時變化。 勾選 使用交通流量時，“連接起始點到目的地”工具會考慮到這些因素；但不是所有的區域都支持交通流量計算。 按一下工具上的 查看可用性連結，以確定您的研究區域是否支持該功能。

結果圖層名稱

在 我的內容中建立並新增到地圖中的圖層的名稱。 預設名稱是根據分析類型以及分析圖層名稱。 如果該名稱的圖層已存在，則系統將要求您提供其他名稱。

使用 將結果儲存到下拉式方塊，可以在 我的內容中指定文件夾的名稱，結果將儲存到該文件夾中。
